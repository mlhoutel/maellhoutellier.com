EXPLANATIONS

Dash Raider is a game we designed with friends during the special "Human Computer Interaction" week: We created the concept and the game in 5 days.

RULES

Dash Raider is a 2D/3D rogue-lite and puzzle game, where you have to evolve in a maze. To move forward, you have to unlock the gates by activating the levers and beating enemies, but you have a limited number of actions and lives.

DETAILS

We made 100% of the game's ressources with various softwares:

  • 3D objects (voxels): MagicaVoxel
  • Musics: Bosca Ceoil
  • Game engine: Unity/C#

EXPLANATIONS

Chroma Blast is the first group project we did at the IUT. With 3 others students, we created the game in 4 days. We made the report and present the final version in front of an amphitheater of 120 persons.

RULES

It's a Rythm Game/Space-Shooter. You have to shoot the right enemy at the right time: only when music block and spacialship of the same colour are aligned. You have a defined time to beat the enemies and collect the more points you can. But if you don't press the shoot button at the right time or if you shoot the enemy with the wrong color, you loose points. There is 3 levels of difficulty.

DETAILS

We used these softwares to create the game:

  • Graphics and Animations: Krita/PS
  • Graphic Library: SFML
  • Programming Language: C/C++
  • IDE: Code::Blocks

Fractals have always fascinated me! Thanks to extremely simple rules, the resulting shapes are complex and amazing...

I drew these fractals with differents methods, according to the type of fractal:

Conways Game

Short animations of the Conways game

Short animations of the Conways game

EXPLANATIONS

Conways Game is pretty similar to the fractals on some points: with a very simple mecanism, this system can create some strange structures, but unlike fractales, it have an other dimention: the time. Thanks to this, you can use it to simulate strange bacterian ecosystems, and the process is Full Turing. I made a grid where you can manually draw and erase the cells you don't want, and can change many things in the parameters.

RULES

Each pixel is a living cell that can die, according to these rules:

Births: Each dead cell adjacent to exactly three live neighbors will become live in the next generation.

Death by isolation: Each live cell with one or fewer live neighbors will die in the next generation.

Death by overcrowding: Each live cell with four or more live neighbors will die in the next generation.

Survival: Each live cell with either two or three live neighbors will remain alive for the next generation.

DETAILS

Language used to generate the Conways game:

  • Conways Game: C# and WPF
